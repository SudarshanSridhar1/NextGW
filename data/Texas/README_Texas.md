# Texas (ERCOT) Subsystem — Files & Usage

Derived from the full USA synthetic test system (`case.mat`), covering the
**8 ERCOT zones (301–308)** of the Texas interconnect.

---

## Source Data

| File | Description |
|------|-------------|
| `case.mat` | Full USA synthetic grid (MATPOWER mpc format): 82,071 buses, 14,050 generators, 104,192 branches across Eastern, Western, and Texas interconnects |
| `bus.csv` | Bus-level data including `bus_id`, `Pd` (base load in MW), `zone_id` |
| `demand.csv` | Hourly zone-level load timeseries (8,784 hours, year 2016, UTC). Columns are zone IDs; Texas zones are `301`–`308` |
| `zone.csv` | Zone metadata: zone ID, name, state, interconnect, timezone |
| `plant.csv` | Generator data including `bus_id`, `Pmax`, `Pmin`, fuel type |

## Generated Files

### `case_texas.mat`

MATPOWER mpc struct containing only the Texas subsystem. Generated by `extract_texas.py`.

| Property | Value |
|----------|-------|
| Buses | 2,000 |
| Generators | 600 (598 original + 2 DC tie replacements) |
| Branches | 3,206 AC branches |
| DC lines | 0 (2 original DC ties to Eastern dropped) |
| Slack bus | 1 (pre-existing in Texas subset) |
| Format | MATPOWER v2 mpc (loadable via `scipy.io.loadmat`) |

**DC tie treatment:** The original system has 2 DC lines connecting Texas to the
Eastern interconnect (−200 MW and −600 MW scheduled). Since ERCOT has no AC
connections to other interconnects, these are dropped and replaced with fixed
generators on the Texas-side buses to preserve power balance.

**Capacity vs. demand sanity check:**

| Metric | Value |
|--------|-------|
| Total Pmax | 110,361 MW |
| Total Pmin | 25,733 MW |
| Base case Pd (bus.csv) | 67,109 MW |
| Peak demand (timeseries) | 71,093 MW |
| Min demand (timeseries) | 25,107 MW |
| Pmax / Peak demand | 1.55× |

Generator fuel mix (Pmax): natural gas 68,642 MW · wind 19,063 MW · coal 14,502 MW ·
nuclear 5,139 MW · solar 2,460 MW · hydro 555 MW.

### `texas_demand.csv.gz`

Hourly bus-level load timeseries for all 2,000 Texas buses, gzip-compressed.
Generated by `make_texas_demand.py`.

| Property | Value |
|----------|-------|
| Rows | 8,784 (hourly, 2016-01-01 00:00 UTC to 2016-12-31 23:00 UTC) |
| Columns | 1 timestamp + 2,000 bus columns |
| Units | MW (rounded to 1 decimal place) |
| File size | ~19 MB (compressed) / ~78 MB (uncompressed) |

**Column layout:**

```
UTC Time | 3001001 | 3001002 | 3001003 | … | 3008xxx
```

**Reading the file:**

```python
import pandas as pd

df = pd.read_csv("texas_demand.csv.gz", compression="gzip", index_col="UTC Time")
# df.shape → (8784, 2000)
# df.index → DatetimeIndex (UTC)
# df.columns → bus IDs as strings, e.g. '3001001', '3001002', …
```

**Disaggregation method:** Each bus receives a fixed fraction of its zone's
hourly demand, proportional to its base-case `Pd` from `bus.csv`:

```
bus_demand[t] = zone_demand[t] × (bus_Pd / zone_total_Pd)
```

Buses with `Pd = 0` (875 out of 2,000) receive a zero timeseries. The sum of
all bus columns at any timestep equals the sum of the 8 zone columns in
`demand.csv` for that hour.

### `ptdf.py`

Python module to compute the DC **Power Transfer Distribution Factor (PTDF)**
matrix from any MATPOWER `.mat` case.

**What is PTDF?**
`PTDF[l, i]` is the fraction of 1 MW injected at bus `i` (and withdrawn at the
slack bus) that flows on branch `l`. Values lie in `[−1, 1]`. It is derived
from the DC power flow approximation (lossless, linear) and is used for
congestion analysis, sensitivity studies, and locational marginal price
calculations.

DC lines are excluded: they carry fixed scheduled power and are not governed by
network physics.

**Usage:**

```python
from ptdf import compute_ptdf

ptdf, bus_ids, f_buses, t_buses = compute_ptdf("case_texas.mat")
# ptdf  : (3206, 2000) float64
# bus_ids : (2000,) int  — bus numbers, matching ptdf columns
# f_buses : (3206,) int  — from-bus of each active branch
# t_buses : (3206,) int  — to-bus  of each active branch
```

**Algorithm:**

1. Build branch susceptances `b = 1 / (x × tap)`
2. Build `Bf` (n_branch × n_bus): `Bf[l, f] = b[l]`, `Bf[l, t] = −b[l]`
3. Build `Bbus` (n_bus × n_bus): DC nodal admittance matrix
4. Remove slack row/col → `B_red` (1999 × 1999)
5. Solve `B_red.T @ PTDF_ns.T = Bf_ns.T` via sparse LU factorisation
6. Insert zero column for slack bus → full `(3206, 2000)` PTDF

**Sanity checks (run `python ptdf.py`):**

| Check | Expected |
|-------|----------|
| All values in `[−1, 1]` | Physical bound |
| Slack column all zero | No flow from self-injection |
| Solve residual `< 1e-6` | Numerical accuracy |
| Radial bus `\|PTDF\| = 1.0` | 100% of injection flows through its single branch |

## Scripts

### `extract_texas.py`

Extracts the Texas subsystem from `case.mat` and writes `case_texas.mat`.

```bash
python extract_texas.py
```

### `make_texas_demand.py`

Disaggregates zone-level demand from `demand.csv` to bus level and writes
both `texas_demand.csv` and `texas_demand.csv.gz`.

```bash
python make_texas_demand.py
```
