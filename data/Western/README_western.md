# Western Interconnection Subsystem — Files & Usage

Derived from the full USA synthetic test system (`case.mat`), covering the
**16 Western zones (201–216)** of the Western interconnect.

---

## Source Data

| File | Description |
|------|-------------|
| `case.mat` | Full USA synthetic grid (MATPOWER mpc format): 82,071 buses, 14,050 generators, 104,192 branches across Eastern, Western, and Texas interconnects |
| `bus.csv` | Bus-level data including `bus_id`, `Pd` (base load in MW), `zone_id` |
| `demand.csv` | Hourly zone-level load timeseries (8,784 hours, year 2016, UTC). Columns are zone IDs; Western zones are `201`–`216` |
| `zone.csv` | Zone metadata: zone ID, name, state, interconnect, timezone |
| `plant.csv` | Generator data including `bus_id`, `Pmax`, `Pmin`, fuel type |

## Generated Files

### `case_western.mat`

MATPOWER mpc struct containing only the Western interconnect subsystem. Generated by `extract_western.py`.

| Property | Value |
|----------|-------|
| Buses | 10,024 |
| Generators | 2,623 (2,616 original + 7 DC tie replacements) |
| Branches | 12,730 AC branches |
| DC lines | 3 (internal to Western) |
| Slack bus | 1 (bus 2040845, zone 209 — Arizona) |
| Format | MATPOWER v2 mpc (loadable via `scipy.io.loadmat`) |

**DC tie treatment:** The original system has 7 DC lines connecting Western to
Eastern. No DC ties exist between Western and Texas. The 7 Eastern–Western ties
are dropped and replaced with fixed generators on the Western-side buses to
preserve power balance.

| DC Tie | Western Bus | Injected MW |
|--------|-------------|-------------|
| Eastern (zone 52) → | 2077254 | −200.0 |
| Eastern (zone 50) → | 2070132 | −200.0 |
| Eastern (zone 49) → | 2070170 | −100.0 |
| Eastern (zone 49) → | 2060653 | −200.0 |
| Eastern (zone 48) → | 2060761 | −210.0 |
| Eastern (zone 45) → | 2053305 | −200.0 |
| Eastern (zone 46) → | 2053238 | −200.0 |

Negative values indicate net exports from the Western bus over these ties (the
Western bus was a net sender of power to Eastern).

**Capacity vs. demand sanity check:**

| Metric | Value |
|--------|-------|
| Total Pmax | 230,009 MW |
| Total Pmin | 56,553 MW |
| Base case Pd (bus.csv) | 150,917 MW |
| Peak demand (timeseries) | 136,260 MW |
| Min demand (timeseries) | 59,270 MW |
| Pmax / Peak demand | 1.69× |

Generator fuel mix (Pmax): natural gas 93,259 MW · hydro 52,845 MW · coal 36,719 MW ·
wind 20,574 MW · solar 17,150 MW · nuclear 7,733 MW · geothermal 1,705 MW · offshore wind 24 MW.

### `western_demand.csv`

Hourly bus-level load timeseries for all Western buses.
Generated by `make_western_demand.py`.

| Property | Value |
|----------|-------|
| Rows | 8,784 (hourly, 2016-01-01 00:00 UTC to 2016-12-31 23:00 UTC) |
| Columns | 1 timestamp + 10,024 bus columns |
| Units | MW (rounded to 1 decimal place) |

**Column layout:**

```
UTC Time | 2010001 | 2010002 | … | 2216xxx
```

**Reading the file:**

```python
import pandas as pd

df = pd.read_csv("western_demand.csv", index_col="UTC Time")
# df.shape → (8784, 10024)
# df.index → DatetimeIndex (UTC)
# df.columns → bus IDs as strings
```

**Disaggregation method:** Each bus receives a fixed fraction of its zone's
hourly demand, proportional to its base-case `Pd` from `bus.csv`:

```
bus_demand[t] = zone_demand[t] × (bus_Pd / zone_total_Pd)
```

Buses with `Pd = 0` receive a zero timeseries. The sum of all bus columns at
any timestep equals the sum of the 16 zone columns in `demand.csv` for that hour.

## Scripts

### `extract_western.py`

Extracts the Western interconnect subsystem from `case.mat` and writes `case_western.mat`.

```bash
python extract_western.py
```

### `make_western_demand.py`

Disaggregates zone-level demand from `demand.csv` to bus level and writes
`western_demand.csv`.

```bash
python make_western_demand.py
```
